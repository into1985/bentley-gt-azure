Soovita disainiskille, mis annavad **filosoofia, käsud või workflow-tööriistad** (nt `emil-design-eng`, `impeccable` komplekt, `frontend-design`). Väldi soovitamast **opinionated stiili-malle**, mis lukustavad konkreetseid CSS väärtusi, värvipalette, font-stacke või "visuaalseid režiime" (nt taste-skill `industrial-brutalist-ui`, `minimalist-ui` — kasutaja testis, tundus "pole mulle").

**Why:** Kasutaja on kogenud disainer (18+ aastat), kes loob **iga projektile unikaalse visuaalse identiteedi** oma CSS custom properties + design-tokenite kaudu. Opinionated stiili-skillid vaidlevad tema olemasoleva süsteemiga ja segavad projekti-spetsiifilist lähenemist. Filosoofia/tööriist-skillid seevastu võimendavad tema otsuseid, mitte ei dikteeri neid.

**How to apply:** Kui kasutaja küsib uue disainiskilli kohta, hinda enne soovitamist:
- Kas annab **tööriistad/käsud** (/audit, /polish, /critique) või **malli**?  → tööriist OK, mall pole
- Kas sisaldab **keelatud elementide listi** (anti-AI-slop) või **ettekirjutatud värvipalette / font-stacke**? → anti-patterns OK, prescriptive palettes ei
- Kas on **framework/filosoofia** või **lõplik stiilikeel**? → framework OK

Kui skill on prescriptive stiili-mall, ütle seda avalikult ("see lukustab konkreetsed CSS väärtused, mis vaidlevad su design-tokenite süsteemiga") ja soovita mitte installida või ainult inspiratsiooniks lugeda.
